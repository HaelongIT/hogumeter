#
#   <클래스명>  <docs/91의 열린 Q-ID>  <한 줄 이유>
#
# 규칙(scripts/check-domain-consumers.sh가 강제한다):
#   - 여기 없는 소비처 0 클래스는 차단된다. 순수 도메인은 단위 테스트만으로 GREEN이 되므로
#     **죽어 있어도 아무 신호가 없다.** `effective_interval_with_robots`가 그랬고(robots Crawl-delay가
#     한 번도 안 지켜졌다), `Purchase.expire()`가 그랬다(관찰이 영원히 안 끝났다).
#   - 인용한 Q가 docs/91에서 **열려 있어야** 한다. 해소되면 면제가 만료되고 게이트가 다시 묻는다.
#   - 다시 소비되기 시작하면 여기서 지워야 한다(낡은 면제는 다음 결함을 숨긴다).
