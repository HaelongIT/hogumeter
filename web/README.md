# web — 최소 슬라이스 (React · Vite · TypeScript)

확정본 §7이 **"제품 등록 + 후보 선택 + variant/키워드/목표가 설정 화면은 1차 검증 전에 선개발"** 로 못박은 그 슬라이스에서 출발했다.

거기에 **읽기 전용 판단 화면**과 **구매 기록(PUR)** 을 앞당겨 붙였다 — 확정본상 M4·M5지만, M1의 잔여 블로커가 전부 외부(토큰·키·승인)라 코드로 갈 수 있는 최고가치 작업이었고 core 변경이 0이라 되돌리기 쉽다(`working-area/decision-log.md` 2026-07-09). **비교·승격 큐 화면은 여전히 후순위.**

```bash
npm install
npm run dev      # http://localhost:5173 — /api 는 :8080(core)로 프록시
npm test         # Vitest
npm run build    # 타입체크(tsc --noEmit) + 프로덕션 빌드
```

## 배포

```bash
docker compose up -d   # http://127.0.0.1:3000
```

nginx가 정적 자산을 서빙하고 `/api`를 `core:8080`으로 프록시한다.

## 지금 되는 것

- **제품 등록**(REG-02): 가격축 **조합**대로 variant 생성(용량 2 × 색상 2 → 4개). 등록 전에 "생성될 variant N개"를 미리 보여준다. 별칭·수요축 모드(GROUPED/SPLIT).
- **등록된 제품 목록**: variant와 함께 **`variantId`를 노출**한다 — 기준가·신호·주기·구매 조회가 전부 이걸 요구하는데, 등록 응답은 `productId`만 준다.
- **등록 → 판단 흐름**: 등록에 성공하면 그 제품의 variant를 나열하고 "판단 보기" 버튼을 준다. **여럿이면 골라주지 않는다**(원칙 2).
- **판단 화면**(`decision/`): 신호등(SIG) · 핫딜 기준가/굿딜라인 · 갭 · 딜 주기(CAD)를 한 화면에. 기간 P(3/6/12개월)는 **사용자 손잡이**(원칙 4) — 단 신호등은 core가 6개월로 고정하므로 기간이 다르면 그 사실을 화면에 밝힌다.
- **구매 기록**(`purchase/`, PUR): 실지불가·구매일·관찰 기간을 기록하고, 구매별 관찰 문맥을 한 줄로 보여준다. 날짜만 아는 구매 시각은 **23:59 KST**로 환산해 보낸다(V2 계약: 입력 계층 책임). 표시는 항상 KST(OPS-03).

### 정직성을 순수 함수의 테스트로 못박는다

`decision/present.ts`·`purchase/present.ts`는 도메인 응답 → 문구 변환만 하는 순수 함수다. 렌더링이 아니라 **단위 테스트가** 규칙을 지킨다.

- SPARSE/NONE에선 통계 용어도 금액도 내지 않는다(테스트가 `\d{1,3}(,\d{3})+원` 패턴의 부재를 단언).
- 주기(CAD)에 "예상·예측"을 쓰지 않는다(단어의 부재를 단언).
- **`currentPrice === 0`은 가격이 아니라 미확립 표식**이다(`docs/91` Q-53). core가 갭을 `−100%`로 계산해 보내더라도 그리지 않는다 — 해석은 `CURRENT_PRICE_UNAVAILABLE` 상수 한 곳.
- 관찰 문맥의 세 모드는 배타이고 모드 밖 필드는 `null`이다. `null`을 "0"으로 읽지 않는다.

## 아직 안 되는 것 (정직하게)

- **네이버 후보 검색**(REG-01 주 경로): API 키 미발급(`docs/91` Q-3). 지금은 **수동 입력만** — REG-01이 인정한 폴백이다. 화면에도 그 이유를 밝힌다.
- **알림 정책 설정**(REG-03): `alert_policy` 테이블·엔티티는 있으나 **REST가 없다**. 목표가·기간 P·K_display·제외 키워드·quiet hours 화면은 쓰기 API가 생긴 뒤 — `docs/91` Q-48.
- **접근 통제**(SEC-02): Basic Auth는 **구현됐으나 기본 off**다. `.env`의 `WEB_BASIC_AUTH_HTPASSWD`를 채워야 켜진다(`docker-entrypoint.d/40-basic-auth.sh`). compose는 `127.0.0.1`로만 개방한다. **HTTPS는 아직 없다** — 평문 HTTP에서 Basic Auth는 자격증명을 그대로 노출한다. 공개망 노출 전 `decisions-needed` D-4·`pre-deploy-checklist` §C.
- **비교·승격 큐 화면**: 없다(M3·M4).
- **백필 표기**(REG-04): 백필 자체가 미구현.

## 설계 메모

- **CORS가 필요 없다.** 운영에선 nginx가, 개발에선 Vite 프록시가 `/api`를 core로 넘긴다. 브라우저에겐 동일 오리진이라 교차 출처 요청이 발생하지 않는다 — core에 CORS 설정을 추가할 이유가 없고, 개발과 운영의 동작이 같다.
- **nginx는 core 없이도 뜬다.** `proxy_pass`에 리터럴 호스트를 쓰면 기동 시점에 DNS를 해석하다 실패해 web 전체가 죽는다(정적 페이지조차). 업스트림을 변수로 두고 `resolver`를 지정해 **요청 시점**에 해석한다 — core가 없으면 `/api`만 502다.
- `src/api/types.ts`는 **core 컨트롤러를 직접 읽고 옮긴 계약**이다. 추측하지 않았다.
- 통계 필드가 `null`인 것은 버그가 아니라 **도메인 계약**이다 — 표본이 빈약하면 통계 용어를 쓰지 않는다(절대 원칙 1). 타입이 null을 감추지 않는다.
- `{code, message}`가 아닌 실패(바인딩 오류·500)도 삼키지 않고 `HTTP_{status}`로 살린다.
