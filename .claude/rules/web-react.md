# web (React · Vite · TypeScript) 규율

> 보편 규칙은 CLAUDE.md `## 축적된 규칙`. 여기엔 web 한정만 둔다.
> `web/` 파일을 열 때만 로드된다 — core·collector 작업 시엔 비용 0.

## 계약

- **`src/api/types.ts`는 core 컨트롤러 원문에서 옮긴 것**이다. 추측해서 필드를 만들지 않는다. 계약이 의심되면 `core/src/main/java/.../adapter/web/*.java`와 도메인 record를 직접 읽는다.
- **응답 인터페이스에 필드를 추가하면 `scripts/smoke.sh`의 그 뷰 필드 검증 목록에도 추가한다.** core가 필드 이름을 바꾸면 단위 테스트는 GREEN인데 화면만 조용히 `undefined`를 그린다 — 부품별 GREEN은 모듈 경계 계약을 보장하지 않는다. 스모크가 `BenchmarkView`·`SignalView`·`CadenceView`·`PurchaseObservation`+`ObservationContext`의 전체 필드 존재를 종단 검증한다(정본 = 각 인터페이스).
- **통계 필드의 `null`은 버그가 아니라 도메인 계약**이다 — 표본이 빈약하면 통계 용어를 쓰지 않는다(절대 원칙 1). 타입이 `null`을 감추지 않게 한다(`strict`, `noUncheckedIndexedAccess`).
- **에러는 삼키지 않는다.** core가 `{code, message}`를 주면 `code`를 보존하고, 아니면 `HTTP_{status}`로 살린다. **등록 경로도 이제 `REG_INVALID_PRODUCT`로 400을 준다**(Q-49 해소, 2026-07-21 — 빈 이름·null 축/모드는 500이 아니라 코드). 클라이언트 검증은 여전히 편의일 뿐이다(curl 우회는 서버가 막는다).
- **클라이언트 검증은 방어가 아니라 편의다.** `buildCommand`가 막아도 curl로 직접 치면 통과한다.

## 프록시·인프라

- **CORS를 쓰지 않는다.** 개발은 Vite 프록시, 운영은 nginx가 `/api`를 core로 넘긴다 → 브라우저에겐 동일 오리진. **web 때문에 core 설정을 바꾸지 않는다.**
- **nginx `proxy_pass`의 업스트림은 변수 + `resolver`로 둔다.** 리터럴 호스트를 쓰면 nginx가 기동 시점에 DNS를 해석하다 실패해 **정적 페이지조차 못 뜬다**. 변수를 쓰면 요청 시점에 해석하고 대상이 없으면 502를 준다. (99: 2026-07-09)
- **`depends_on`은 "떴다"만 보장한다.** 없앨 수 있으면 없앤다 — web은 core 없이도 떠야 한다.
- 이미지는 **실제로 띄워서** 확인한다. `docker build` 성공은 기동 성공이 아니다. `bash scripts/smoke.sh`.

## 정직성은 표시 계층에서 강제한다

- **도메인 응답 → 문구 변환은 `decision/present.ts` 같은 순수 함수에 둔다.** 컴포넌트 안에서 포맷하지 않는다 — 정직성 규칙은 렌더링이 아니라 **단위 테스트로** 지켜져야 한다.
- **판정은 히어로다**(2026-07-12) — 이 화면의 답은 신호등 하나이고, 계기·수치는 그 근거다(`.verdict` = 램프 + 헤드라인 + 서브라인, 그 아래 `근거 · 계기`). 히어로의 서브라인(`verdictSubline`)은 **답 바로 아래라 가장 크게 읽히는 근거**다 — 기준가가 설 때만 금액을 낸다(SPARSE/NONE·미확립이면 숫자 없이 이유만).
- **신호색을 드는 것은 램프 하나뿐이다.** 헤드라인은 **중립 잉크** — 다크에서 잉크는 대비 ~15:1, 신호 초록 ~7:1·빨강 ~5:1이라 **색을 넣으면 가독성이 오히려 절반이 된다.** "더 강하게"의 지렛대는 색이 아니라 **크기·굵기**다. 램프 발광 세기는 `--lamp-glow-*` 셋이 유일한 이음새(기본 = 중 — 강하면 다크 화면을 오래 볼 때 눈이 피로하다).
- **SPARSE/NONE에서는 통계 용어도 금액도 내지 않는다.** 테스트로 `/\d{1,3}(,\d{3})+\s*원/`이 없음을 단언한다. 대신 사례를 원문 링크로 나열한다(절대 원칙 2).
- **주기(CAD)에 "예상·예측"을 쓰지 않는다.** 테스트가 그 단어의 부재를 단언한다.
- **`currentPrice === null`이면 미확립이다**(`currentPriceUnavailable`, `docs/91` Q-53). core가 미확립이면 `currentPrice: null`·`gap`의 두 leg도 null로 보낸다(갭을 아예 계산하지 않는다) — 갭을 그리지 않는다. 해석은 **한 곳**에만 둔다. 예전엔 sentinel 0이라 `=== 0`이었으나(그때 core가 `−100%`를 보냈다), 이제 null이 타입상 명시라 각 소비처가 `=== null`로 내로잉한다.

## 범위

- 확정본상 조회·비교 화면은 M4다. **읽기 전용 판단 화면(`decision/`)·미상 큐(`review/`)만 앞당겼다** — M1 잔여 블로커가 전부 외부(토큰·키·승인)이고 core 변경이 0이라 되돌리기 쉽다(`decision-log` 2026-07-09). 승격 큐·비교는 여전히 후순위.
- 네이버 후보 검색(REG-01 주 경로)은 키 미발급(Q-3) → 수동 폴백. **그 이유를 화면에 밝힌다**(과대약속 금지).
- 알림 정책 설정(REG-03)은 `policy/AlertPolicyPanel`이 판단 화면 안에 둔다. **여섯이 저장된다**(목표가·판정 기간·방해금지 2개 + K_display〔Q-48 ①〕 + 제외 키워드〔Q-28〕). 제외 키워드는 쉼표 한 줄로 입력하고 배열로 보낸다 — 걸리는 딜은 기준가·신호·알림 전 통계에서 빠지고, 저장 시점이 아니라 **조회 시점**에 지금 목록으로 판정하므로 이미 들어온 딜에도 소급된다. ⚠️라벨 모드 토글·수요축 필터(demand_axis_filter)는 아직 core 엔티티 미매핑(Q-66). **없는 손잡이를 그리지 않는다** — 그리면 저장되는 줄 안다.
- **미상 큐(`review/`)는 승격·기각이 된다**(Q-15, 2026-07-12). 이상치는 승격(정상 딜로 표본 복귀)·기각(사기·낚시 영구 제외) 둘 다, **미상 항목은 기각만** — 승격하려면 variant를 지정해야 하는데 그 입력이 없어 core가 400으로 막는다. **없는 버튼은 그리지 않는다**(과대약속 금지). 처리 후엔 목록을 다시 불러 항목이 큐에서 내려간 걸 눈으로 보여준다. `occurrences > 1`은 재처리 멱등 부재의 증거였고(Q-27④로 해소 — 이제 한 행에 접혀 카운터로 센다) `firstSeenAt ~ lastSeenAt`이 그 나이다(`seenLine`). **API가 내는 필드를 화면이 안 읽으면 그건 우리가 만든 죽은 필드다.**
- **본문 없는 명령(200 + 빈 본문)은 `client.ts`의 `command()`로 부른다** — `request()`는 항상 `response.json()`을 파싱해 빈 본문에서 터진다.
- **"이 값으로 무엇을 결정할 수 있나"를 물어라.** `후보 2개`·`700,000원`처럼 **개수·숫자만** 그리면 사람은 아무것도 못 고른다 — 후보는 **이름**으로, 이상치는 **대상**(`제품 — variant`)으로 말한다. 못 푸는 값은 지어내지 말고 `대상 미상`·`#999`로 그 사실을 그린다.
- **"…중"이라고 쓰기 전에 그 일을 하는 코드를 찾아라.** `성적 집계 중`은 진행 중이라는 뜻인데 집계하는 코드가 없었다(Q-62) — `관찰 종료 · 성적표는 아직 발급되지 않습니다`로 고쳤다.
- 라우터를 들이지 않는다. 화면이 셋이라 `App.tsx`의 탭으로 족하다 — URL이 필요해지면 그때 넣는다.
- **수요축(demand axis)은 통째로 미구현이다**(`docs/91` Q-66). `buildCommand`는 모든 축을 `axisType: 'PRICE'`로 보내고, `DemandAxisMode.SPLIT`은 저장만 되며 어떤 코드도 그 값으로 분기하지 않는다. 등록 화면이 그 사실을 **문장으로** 말하고 테스트가 그 문장을 요구한다 — 구현되면 그 두 문단만 지운다(seam). **없는 손잡이를 그리지 않는다**가 아니라, 확정본에 있는 손잡이라 지우지 못하므로 **동작하지 않는다고 말한다.**
