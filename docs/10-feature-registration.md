# 10. 기능1 — 제품 등록 (REG)

## REG-01 제품 검색 등록 (주 경로)
- 사용자가 제품명 텍스트 입력 → 네이버 쇼핑 검색 API 호출 → 후보 목록(제목·이미지·카탈로그ID·가격대) 표시 → 사용자가 1개 선택.
- URL 입력은 보조: URL에서 추출 가능한 토큰만 검색어로 프리필. **어떤 경우에도 다나와/네이버 페이지를 서버가 fetch하지 않는다.**
- 수용 기준: "아이폰 17 256" 입력 시 후보가 뜨고 선택으로 Product 생성. API 실패 시 명확한 오류 + 수동 생성 폴백.

## REG-02 축 정의
- 등록 화면에서 priceAxis(값 목록 포함)와 demandAxis를 사용자가 정의. demandAxisMode 기본 GROUPED, 토글로 SPLIT.
- demandAxis 값은 다중 선택. "미지정" 상태 없음(전체 선택 = 전체). **기본 = 전체 값 선택.**
- **미선택 값 처리(v1.3, DN-C6)**: 미선택 값의 딜 = "범위 외"로 **전 집합 제외**(REJECTED 아님 — 재선택 시 편입). GROUPED 부분 선택에서 축값 판별 불가 딜은 잠정 제외 + 승격 큐 + 미산입 공시. **부분 선택은 미상 큐 증가 비용을 화면에서 고지.**
- 수용 기준: priceAxis 값 조합대로 Variant가 생성된다. SPLIT 전환 시 기존 딜은 축값 판별 재시도 후 미상 버킷 처리.

## REG-03 알림 정책 설정
- variant별: targetPrice(선택), 기간 P, K_display(3~10), 제외 키워드(전역 기본셋 상속 + 제품별 추가), quiet hours, 제외 vs ⚠️라벨 토글.
  - **⚠️라벨 통계 지위(v1.3, DN-C5)**: 라벨 모드도 통계엔 **미산입**(모드 무관 전 통계 제외). "제외"와의 차이는 **화면 가시성뿐**(라벨=회색 표시, 제외=숨김) — 오염 표본은 어느 모드든 기준가·집합에 안 들어간다.
- 전역 제외 키워드 기본셋 시드: 벌크, 리퍼, 중고, 해외직구 (+ 제품별 예시: 약정, 번이, 성지, 0원, 요금제)

## REG-04 백필
- 등록 직후 collector에 백필 작업 큐잉. 사이트별 가능 깊이만큼 과거 글 수집(origin=BACKFILL).
- 수용 기준: 백필 딜이 기준가 n에 포함되고, 교차검증 요건이 면제되며, 화면에 backfill 표기.

## 테스트 포인트
- 도메인: Variant 생성 조합 로직, SPLIT 전환 시 재분류 규칙 (순수).
- adapter/naver: WireMock으로 검색 응답 → 후보 DTO 매핑.
- web: 등록 플로우 컨트롤러 슬라이스 테스트.
