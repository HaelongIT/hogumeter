# 14. 기능5 — 중고 (USED) — 개발 2순위 (M2)

## USED-01 조건검색 (3계층)
- required(AND, 전부 매칭) / bonusGroups(OR 그룹, 그룹별 SORT|TRIGGER 모드) / exclude(NOT, 전역+검색별+사후학습). 동의어는 사용자가 그룹 내 나열 — 내장 동의어 사전 만들지 말 것.
- 기준가 합성 금지. 중고의 가격 맥락은 스냅샷(무가공 나열)만.

## USED-02 번개장터 폴링
- 검색 목록 페이지만 주기 방문(기본 10분, 검색별 조정). 목록에서 신규/가격변동/소실(SOLD 추정)/끌올(listingId dedupe) 감지. 부하 O(검색 수).
- 상세 fetch는 알림 승격 시 1회(설명 확보). 본문만 수정(가격 불변)은 미추적.
- 알림: required 통과 + (TRIGGER 그룹 충족) + targetPrice 이하 → 발송. SORT 그룹 매칭은 배지·정렬만. 가격변동·SOLD 후속은 promoted 매물 한정.

## USED-03 당근 (반자동)
- 감지: 당근 네이티브 키워드 알림에 위임(시스템 외부). 자동 크롤링·fetch 파이프라인 구축 금지.
- 평가기 입력 3단: URL fetch 1회 시도 → 실패 시 "텍스트 붙여넣기 요청" → 수동 필드 입력. 입력 계층은 URL|TEXT|MANUAL 추상화로 처음부터 설계.

## USED-04 URL 평가기 (플랫폼 무관 매물 판단 입구)
출력 3종:
1. 가격 맥락: 신품 기준가 대비 % + marketSnapshot(출처 "번개장터 활성 매물" 상시 표기, 평가 대상 위치 표시)
2. 위험 신호: exclude/업자 레퍼토리 키워드 히트 + "스냅샷 최저 대비 X% 이상 저렴" 플래그. 신호 나열만, 판정 문구 금지.
3. 원문 링크 + 구조화 필드(가격·지역·등록일)
- 추출 로직은 인터페이스로 분리(v1 규칙 기반, v2 LLM 교체 가능 지점).

## USED-05 메모·병렬 비교
- 자유 메모(listing_note) + 제품별 comparison_axis 정의 + 값 승격(listing_axis_value).
- 병렬 비교표: 상단 축 정렬(빈칸 노출 = 미확인 체크리스트), 하단 매물별 메모 전문.

## 테스트 포인트
- 3계층 필터: AND/OR(SORT vs TRIGGER)/NOT 조합 매트릭스, 동의어 그룹, 대소문자·띄어쓰기 정규화 (순수).
- 생애주기: 목록 diff → 신규/변동/SOLD/끌올 판정 (순수, 목록 스냅샷 2개 입력).
- 평가기: 입력 3단 폴백 분기, 위험 신호 룰.
- bunjang 파서: golden fixture.
