# 00. 개요 & 용어집

## 시스템 목적
사용자가 등록한 제품에 대해 (1) 핫딜 커뮤니티를 감시해 좋은 딜을 즉시 알리고, (2) 과거 핫딜 분포로 "핫딜 기준가"를 산출해 현재가와의 갭을 제시하며, (3) 구매 결심 시점에 판매처를 횡비교하고, (4) 중고 매물을 조건검색·알림·판단보조한다. 사용자는 1인(운영자 본인).

## 용어집 (코드 네이밍에 이 용어를 사용)
| 용어 | 영문(코드) | 정의 |
|---|---|---|
| 제품 | Product | 사용자가 등록한 canonical 제품 |
| 변형 | Variant | 가격결정 축 조합 단위. 분포·기준가의 기본 단위 |
| 가격결정 축 | priceAxis | 제조사가 가격을 달리하는 옵션(용량 등). variant 분리 필수 |
| 수요 축 | demandAxis | 정가 동일하나 시세가 갈리는 옵션(색상 등). 기본 묶음, 사용자 토글로 분리 |
| 딜 | DealEvent | 병합·정규화된 핫딜 1건 (여러 사이트 글 = 1 DealEvent) |
| 딜 원문 | DealPost | 크롤링된 개별 게시글 (DealEvent에 N:1) |
| 핫딜 기준가 | benchmarkPrice | 유효 딜 분포의 median |
| 굿딜 라인 | goodDealLine | 교차검증 딜의 P25 |
| 갭 | gap | 현재가 − 기준가 |
| 교차검증 | crossVerified | 2개 이상 사이트에서 확인된 딜 |
| 미상 | unclassified | 제품/variant/축값 배정 불가 상태 (버킷 격리) |
| 승격 큐 | reviewQueue | 사람이 확정/기각하는 대기열 (미상 분류·이상치 승격·키워드 제안 통합) |
| 이상치 | outlier | 분포에서 튄 값. 방향 구분: upper(제외·무알림) / lower(대박딜: 알림 최우선·기준가 제외) |
| 현재가 | currentPrice | 네이버 API 최저가. 온디맨드 + 1h 캐시 |
| 중고 검색 | UsedSearch | Product 종속. 3계층 키워드 + 목표가 + 폴링 주기 |
| 매물 | Listing | 중고 개별 매물 (1점물) |
| 스냅샷 | marketSnapshot | 현재 활성 매물 가격의 무가공 나열 (통계 합성 아님) |
| 백필 | backfill | 등록 시점 과거 딜 소급 수집. origin=backfill |

## 핵심 수치 (확정)
- K_display: 사용자 설정 3~10, 기본 5 ("기준가" 라벨 최소 표본)
- K_fill = max(7, K_display + 2) (기간 자동확장 목표, 시스템 연동식)
- 표본 판정 단위 = n (전체 유효 딜: 단일사이트·백필 포함, 이상치 제외). m(교차검증)은 표시 전용
- 자동확장 과거 상한 = 12개월
- 핫딜 게시판 폴링 = 게시판당 1req/min / 중고 목록 폴링 기본 10분(제품별 조정)
- 현재가·기능4 비교 캐시 = 1시간
- 딜 병합 = 동일 variant + 가격 ±α + 시간 윈도우(24~48h, 수치는 M0에서 확정)

## 표본 3단 표시 규칙 (정직성 원칙의 구현)
- 충분(n ≥ K_display): benchmarkPrice + goodDealLine 정식 표시
- 빈약(1 ≤ n < K_display): 통계 용어 출력 금지. "최근 핫딜 사례" 나열만. 알림 잣대 = 보유 최저가 + "표본 N건, 참고용" 딱지
- 없음(n=0): 현재가만 + "관측 시작" 표시. 대박딜 폴백 = 현재가 대비 대폭(수치 M0 확정) 하락 시 참고용 딱지 알림
