# used · 00. 개요

## 목적 / 범위
기능5(USED) — 중고 매물의 **정밀 조건검색 + 목표가 알림 + 판단 보조**. 개발 2순위(M2).
신품(BM)과 근본이 다르다: **기준가를 합성하지 않는다** — 중고는 상태 편차(S급·부품용·침수) 때문에
분포 합성이 무의미하다. 대신 3계층 키워드로 정밀하게 거르고, 현재 활성 매물을 **무가공으로 나열**해
사람이 1초에 판단하게 돕는다(절대 원칙 2: 판단은 사람, 시스템은 근거).

- 이 문서 묶음의 경계: `docs/14-feature-used-market.md`(USED-01~05)의 상세화. 정책 충돌 시
  `docs/90-planning-final.md` §5가 최종 권위.
- 걸치는 컴포넌트: **collector**(번개 목록 폴링·파싱) · **core**(3계층 필터·목록 diff 생애주기·평가기
  추출·메모/축, 전부 도메인) · **web**(3계층 검색 등록·평가기 3단 입력·병렬 비교표). 알림 발화 자체(AL)와
  제품 등록(REG)은 별도 모듈 — 여기서는 이음새만 정의.
- 저장: PostgreSQL 단일. **축·메모는 EAV 정식 테이블**(JSONB 남용 금지 — JSONB는 크롤링 원본 보관 전용).
- 마이그레이션: used 스키마는 **V3+**로 core가 TDD로 추가한다(V2 슬롯은 PUR이 소진 — `docs/91` Q-4).

## 기능 목록

| 기능ID | 이름 | 한 줄 |
|---|---|---|
| USED-01 | 조건검색(3계층) | required(AND) / bonusGroups(OR, SORT\|TRIGGER) / exclude(NOT). 내장 동의어 사전 금지 — 사용자가 그룹에 나열 |
| USED-02 | 번개장터 폴링 + 생애주기 | 검색 목록 페이지만 주기 방문(기본 10분). 목록 diff로 신규·가격변동·판매완료·끌올 감지 |
| USED-03 | 당근(반자동) | 감지는 당근 네이티브 키워드 알림에 위임(자동 크롤링 금지). 판단은 URL 평가기로 |
| USED-04 | URL 평가기 | 플랫폼 무관 매물 판단 입구. 입력 3단(URL\|TEXT\|MANUAL), 출력 3종(가격맥락·위험신호·원문+구조화) |
| USED-05 | 메모·병렬 비교 | 자유 메모 + 제품별 비교축 정의 + 값 승격. 병렬 비교표(빈칸=체크리스트) |

## 핵심 개념

- **3계층 필터**(USED-01): 번개 매물의 `title` 문자열을 세 계층으로 판정하는 **순수 함수**.
  - **required (AND)** — 전부 매칭돼야 후보. 제품 정체성(`아이폰17`·`256`).
  - **bonusGroups (OR 그룹)** — 그룹별 모드: **SORT**(있으면 상단·배지, 없어도 알림) / **TRIGGER**(그룹 중
    1개 이상 있어야 알림). 동의어는 사용자가 그룹 안에 직접 나열(`S급, 에스급, 민트`) — 내장 사전 없음.
  - **exclude (NOT)** — 전역 기본값(`부품용·액정파손·침수·삽니다·매입`) + 제품별 + 사후학습(신품과 동일 메커니즘).
- **생애주기**(USED-02): **연속된 두 목록 스냅샷의 diff**로 신규·가격변동·판매완료(목록 소실)·끌올(listingId
  dedupe)을 판정하는 **순수 함수**. 매물 개별 재방문이 아니라 검색 목록 페이지 한 번으로 전부 감지(부하 O(검색 수)).
- **알림**(USED-02): `required 통과 + (TRIGGER 그룹 충족) + targetPrice 이하` → 발송. SORT 그룹 매칭은 배지·정렬만.
  **가격변동·판매완료 후속 알림은 승격된(promoted) 매물 한정**(스냅샷 전체엔 미적용 — 스팸 방지).
- **URL 평가기**(USED-04): 플랫폼 무관. 입력 3단 폴백(URL fetch → 텍스트 복붙 → 수동), 출력 3종.
  추출 로직은 **인터페이스로 분리**(v1 규칙, v2 LLM 교체 지점).
- **메모·축**(USED-05): 자유 메모 + 제품별 비교축 + 메모→축 값 승격. **EAV 3테이블**(JSONB 금지).

## 절대 원칙 연결

- **기준가 합성 금지**(원칙 1 정직성): 중고 가격 맥락은 **스냅샷 무가공 나열**만. median·P25를 만들지 않는다.
  화면에 "비교 대상: 번개장터 활성 매물" 출처를 상시 표기(당근 매물 평가 시 직거래/택배 맥락 차이 인지용).
- **판단은 사람**(원칙 2): 위험 신호는 **나열만**, 판정 문구 금지("사기다"·"안전하다"를 쓰지 않는다).
- **플랫폼 잣대**(원칙 5): 번개=풀 자동(잣대 ③ 공개 목록 수용) / 당근=반자동(잣대 ② IDC 차단·약관 금지 →
  **자동 fetch 시도 금지**, 네이티브 알림 위임) / 중고나라=v1 제외(네이버 카페 구조).
- **과대약속 금지**(원칙 6): 번개 status 코드표 미실측 구간은 잠정 SOLD_OUT으로 두되 그 사실을 표시(`docs/91` Q-44).
