# 13. 기능4 — 구매 시점 판매처 비교 (CMP) — 개발 3순위 (M3)

## CMP-01 온디맨드 비교
- 사용자가 variant 지정 → 네이버 API로 판매처별 최저가 조회(여러 몰 우산) + 쿠팡 데이터(확장 유입분) + 최근 핫딜가(간접) 병렬 표시. 캐시 1h.
- 비교 기준 통일: 실결제가 + 배송비. 같은 화면에 기준가 갭 병기("어디서 + 지금 사도 되나").
- 약속 수위: 페이지 노출 공개 혜택까지만 자동. 장바구니 쿠폰·카드 청구할인은 "직접 확인" 직링크. 완전 자동 체감가를 구현하려 하지 말 것.

## CMP-02 쿠팡 크롬 확장
- 사용자가 쿠팡 상품 페이지를 열면 확장이 가격(와우가/일반가 모두)·배송·노출 혜택을 읽어 core REST로 전송(토큰 인증). 서버발 쿠팡 접근 절대 금지.
- 와우가/일반가 둘 다 저장·표시 + 보기 필터.
- 확장 권한 최소화: 쿠팡 도메인 한정, 읽기 전용 DOM.
- 폴백: 확장 부재/고장 시 반자동 붙여넣기 입력(웹 폼).
- 파트너스 API는 사용하지 않는다(확정 제거).

## 테스트 포인트
- 비교 정렬·정규화(배송비 합산, 와우 필터) 순수 로직.
- extension ingest 엔드포인트: 스키마 검증, 토큰 인증, 멱등.
- 확장 자체: 쿠팡 fixture HTML 대상 파싱 단위 테스트(셀렉터 깨짐 감지용).
