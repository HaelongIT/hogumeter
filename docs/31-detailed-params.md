## 상세 근거

### 1. `MERGE_PRICE_TOLERANCE` = ±2% (하한 ±5,000원)
- **역할**: DealEvent 병합 조건의 가격 축(`docs/90` §딜 병합). 동일 variant·윈도 내에서 두 글의 가격 차가 이 폭 이내면 같은 딜로 병합.
- **왜 비율+하한 혼합**: 고가(100만원대 폰)는 2%면 2만원 — 카드할인 표기 차 흡수에 적당. 저가(1~2만원 소모품)는 2%가 너무 좁아 같은 딜이 갈라짐 → 절대 하한 ±5,000원.
- **되돌리기**: 좁히면 과분할(같은 딜이 여러 건 → n 부풀림), 넓히면 오병합(다른 딜 흡수 → median 왜곡). **의심 시 좁게** 시작이 정직성에 안전하나, 재현율 우선 원칙상 중도값 제안. 1차 검증에서 관측.

### 2. `MERGE_WINDOW_HOURS` = 48h
- **역할**: 병합의 시간 축. docs/90은 "24~48h"로 범위만 확정.
- **왜 48h**: 사이트별 게시 시차·주말 지연 수용(재현율 우선, 놓침>오알림). 미상 잠정 병합도 동일 48h.
- **되돌리기**: 24h로 좁히면 교차검증(m) 놓칠 위험↑. 48h가 보수적(더 많이 묶음)이나 소급 알림이 없어 과병합의 실피해가 작음.

### 3. `OUTLIER_IQR_MULTIPLIER` = 1.5
- **역할**: BM-05 양방향 이상치 판정. Q1−1.5·IQR 미만=LOWER(🔥+제외), Q3+1.5·IQR 초과=UPPER(제외·무알림).
- **왜 1.5**: Tukey 표준. 제품군 특화 로직 없음(docs/90 명시). **주의**: SPARSE(n<5) 구간은 IQR 자체가 불안정 → 이 구간은 BM-05 AC-5 "현재가 기준 폴백 컷"으로 대체(IQR 미적용).
- **되돌리기**: 낮추면(예 1.0) 과민 → 진짜 딜을 이상치로 오제외. 높이면 둔감. 1.5에서 시작해 오제외 사례를 reviewQueue로 관측.

### 4. `COLDSTART_JACKPOT_RATIO` = 0.30
- **역할**: 표본 NONE(n=0) 구간에서 "기준 미확립·참고용" 알림을 낼 유일 조건 — 현재가 대비 30% 이상 싼 딜.
- **왜 30%**: docs/90 가안 유지. 기준가가 없는데 알림을 내는 것이므로 **높은 문턱**이 정직. 낮추면 근거 없는 알림 남발.
- **되돌리기**: 관측 후 조정. 1차 검증 제품이 콜드스타트라 실측 기회 많음.

### 5·6. `K_DISPLAY`=5(3~10), `EXPAND_LIMIT_MONTHS`=12
- docs/90 §8 설정값 장부의 확정값 **재확인**(신규 제안 아님). K_DISPLAY는 사용자 손잡이, EXPAND_LIMIT은 시스템 고정. 카테고리별 상한 조정은 Phase 2.
