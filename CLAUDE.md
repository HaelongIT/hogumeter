# CLAUDE.md — 구매 의사결정 보조 시스템 (핫딜 가격 인텔리전스)

이 파일은 프로젝트의 헌법이다. 모든 세션에서 이 파일을 기준으로 작업한다.
상세 요구사항은 `/docs`에 있으며, 본 파일과 충돌하면 **docs/90-planning-final.md(기획 확정본)가 최종 권위**다.

## 프로젝트 한 줄 정의
핫딜 커뮤니티 데이터로 "핫딜 기준가"를 산출하고, 현재가와의 갭으로 "지금 사도 호구가 아닌지"를 알려주는 **1인용** 구매 의사결정 보조 시스템.

> **현재 = M1(신품 코어 루프) 진행 중.** M0 완료. 수집→적재→파이프라인 트리거→기준가→web 판단 화면까지 `docker compose up -d` 하나로 흐른다(`scripts/smoke.sh`가 매번 종단 검증). 기준가 엔진 정본은 `docs/benchmark/00~07`.
> M1을 막는 것: 텔레그램 토큰(Q-20)·네이버 키(Q-3)·실 폴링 승인·D-3 결정, 그리고 **코드 안의 블로커 Q-27 ③**(품절 딜 오알림 — 토큰 켜기 전 필수).
> **최신 상태·한계는 항상 코드·git·`docs/91-open-questions.md`·`working-area/progress-log.md`가 정본.** 이 문단도 낡을 수 있다.

## 절대 원칙 (모든 코드 결정의 잣대)
1. **정직성**: 표본 빈약 시 통계 용어 금지. 기준가는 항상 "n건(교차 m건)" 동반.
2. **판단은 사람, 시스템은 근거**: 사기/최종 판단 로직을 만들지 않는다. 원문 링크·신호를 모아줄 뿐.
3. **속도·재현율 우선**: 놓침 > 오알림. 애매하면 "후보"로 올려 사람이 1초 확인.
4. **손잡이 배분**: 표시를 바꾸는 설정은 사용자에게, 데이터 진실을 바꾸는 규칙은 시스템 고정.
5. **플랫폼 잣대**: 공식 API 우선 / 기술적 차단은 우회 금지(프록시·UA위장 금지) / 차단 없는 공개 페이지는 저빈도·개인용 수용.
6. **과대약속 금지**: 자동으로 못 잡는 것은 솔직히 경계 긋고 원문 링크로 넘긴다.

## 스택 & 컴포넌트 (확정, 변경 금지)
- **core**: Spring Boot 4.x (Java 21) — 기준가 엔진·매칭·알림·REST API. 상시 가동. 빌드: Gradle(Kotlin DSL) + JUnit 5 + Testcontainers. (D-2: 3.5.x OSS 지원 종료 임박 → 4.1.0 이관, decision-log 참조)
- **collector**: Python 3.12 별도 컨테이너 — 핫딜 3사(뽐뿌/루리웹/펨코) + 번개장터 폴링. krepe90/user-hotdeal-bot 골격 최대 재활용. 도구: uv + pytest.
- **DB**: PostgreSQL 16 단일. Flyway 마이그레이션(core 단독 소유). JSONB는 크롤링 원본 보관 전용.
- **알림**: 텔레그램 봇 (+ 인라인 버튼 승격 액션).
- **web**: React — 제품 등록 + **읽기 전용 판단 화면**(신호등·기준가·갭·주기) + **구매 기록(PUR)**. 비교·승격 큐는 후순위. 빌드: Vite + TypeScript.
- **extension**: 크롬 확장(쿠팡 리더) — 기능4 단계에서만.
- **배포**: Docker Compose, AWS EC2. AWS 종속 관리형 서비스에 코어 로직 금지(이식성). CI: GitHub Actions(테스트 실행).

## TDD 규율 (필수)
- **Red → Green → Refactor.** 실패하는 테스트 없이 프로덕션 코드를 작성하지 않는다.
- 순수 도메인(기준가 산식·이상치 판정·매칭·알림 판정·3계층 필터)은 **IO 없는 순수 함수/클래스**로 작성해 단위 테스트만으로 검증 가능해야 한다. 상세: `docs/21-tdd-guidelines.md`
- 파서는 **fixture HTML(golden file)** 기반 테스트. 실제 네트워크 호출 테스트 금지(스파이크 제외).
- 커버리지는 목표가 아니라 결과다. 단, 도메인 모듈은 뮤테이션에 가까운 엄밀함을 지향.

## 교훈 축적 프로토콜 (필수, 잊지 말 것)
- TDD 사이클·디버깅·스파이크에서 얻은 교훈은 **즉시 `docs/99-lessons.md`에 append**한다. 형식: 날짜 / 맥락 / 증상 / 원인 / 규칙화된 교훈 / 관련 테스트.
- 교훈이 **반복 적용 가능한 규칙**이면 2계층으로 승격한다(중복 금지, 최신 우선) — **보편 규칙**은 본 CLAUDE.md `## 축적된 규칙`에, **언어·디렉토리 한정 규칙**은 `.claude/rules/<scope>.md`에(`paths:` frontmatter를 달면 해당 파일을 열 때만 로드돼 컨텍스트를 아낀다).
- 외부 사이트 구조·API 응답의 실측 발견(파싱 셀렉터, 쿼터 실동작, 차단 징후)은 `docs/98-field-notes.md`에 사이트별로 기록한다. 다음 세션의 나를 위한 문서다.
- 세션 시작 시 99-lessons / 98-field-notes를 반드시 읽고 시작한다.
- 열린 보드(결정·기술보류)와 **기록 드리프트 지표**(교훈 수 vs 축적된 규칙 수)는 `SessionStart` 훅(`.claude/hooks/session-brief.sh`)이 매 세션·매 압축 후 자동 주입한다. 지침은 컨텍스트일 뿐 강제가 아니므로, 승격 누락이 눈에 보이게 만든 장치다.

## 축적된 규칙
> 보편 규칙만 여기 둔다. 언어·디렉토리 한정 규칙은 `.claude/rules/<scope>.md`(paths frontmatter). 출처는 `docs/99-lessons.md`.

### 사실을 다루는 법
- **로드베어링한 사실(경로·타입·필드명·설정 키·문법)은 원문 또는 스키마로 확인한 것만 쓴다.** 서브에이전트·요약 모델의 주장은 가설이다. **우리가 `docs/98`·`99`에 적어둔 "실측" 기록도 가설이다** — 문서가 작업을 막고 있다면 막기 전에 그 근거를 한 번 재현한다.
- **문서와 실행 가능한 계약(테스트·DDL·CHECK)이 모순이면 실행되는 쪽이 진실이다.** 산문은 GREEN일 수 없다. 모듈 경계를 넘는 계약은 한쪽이 소유하고 다른 쪽은 **원본을 참조**한다 — 테스트용 미러는 드리프트하고, 드리프트한 미러는 GREEN인 채로 거짓말한다. **부분 참조도 미러다**: 원본 하나를 지목하는 순간 그 지목이 스냅샷이 된다. 범위를 열고(`V*__*.sql` 전부) 정렬 규칙을 명시한다.
- **틀린 단정을 했으면 즉시 정정한다.** 잘못된 확신은 침묵보다 해롭다.
- **바이트가 의미인 파일은 저장소가 바이트를 못박는다**(`.gitattributes`의 `-text`). 줄끝을 정하지 않으면 각자의 `core.autocrlf`가 정하고, "저장소가 진실"이라는 전제가 조용히 무너진다 — golden을 Windows와 CI가 다른 바이트로 읽는다. 실행되는 것(`*.sh`·훅)은 `eol=lf`.
- **"값 없음"을 값으로 표현하지 않는다.** 스텁·기본값의 sentinel(`0`, `""`)은 스텁 안에 머물지 않고 산식·알림·화면까지 흘러 **정상 응답 모양의 거짓말**이 된다(Q-53: 현재가 0 → "100% 싸다"). 불가피하면 해석을 **한 곳**에 가두고 그 사실을 보드에 적는다.
- **관측 출력은 산문이 아니라 이벤트(JSON, ASCII 이스케이프)로 낸다.** 문구를 grep하는 테스트는 문구를 굳히고, 콘솔 인코딩·파싱·표현이 한 덩어리가 된다. 카운터에서 0을 생략하지 않는다 — "성공했는데 0건"이 사라진다.

### 테스트를 짜는 법
- **부품별 GREEN은 계약을 보장하지 않는다.** 값이 부품 사이를 흐르면 그 경로를 관통하는 좁은 테스트를 하나 둔다(`B(A())`). 소비처의 허용집합은 손으로 만든 값이 아니라 **생산자가 실제로 낼 수 있는 값 전부**로 시험한다.
- **읽기만 하는 테이블·큐·포트를 보면 "누가 여기에 쓰는가"를 프로덕션 코드에서 찾아 이름을 댄다.** 못 대면 그 기능은 죽어 있다 — 테스트의 `save(...)`나 fake의 반환값은 생산자가 아니다. `alert_policy`는 읽히기만 해서 목표가 알림이 발화할 수 없었고, 트리거가 없어 `raw_deal_post`는 소비되지 않았으며, 모든 테스트는 GREEN이었다.
- **결함을 찾았으면 그 거울상을 찾는다.** "읽기만 하는 포트"를 찾았으면 다음은 "쓰기만 하는 포트"다(`review_queue_item` — 쌓이는데 볼 수가 없었다). **소비자를 만드는 일은 생산자를 검사하는 일이다** — 큐를 읽는 순간 큐가 어떻게 채워지는지 드러난다. 드러난 중복·유실을 조용히 정리하지 말고 **세어서 노출**한다(접기만 하고 세지 않으면 결함이 사라진 것처럼 보인다).
- **모듈 테스트가 전부 GREEN이어도 스택은 안 뜬다.** 컨테이너 경계·포트·프록시 DNS·argv 인코딩은 프로세스 밖의 계약이라 어떤 단위 테스트도 안 본다. 종단 스모크를 스크립트로 만들고 CI에 건다.
- **차단·필터 장치는 "무엇을 통과시켜야 하는가"를 먼저·더 많이 테스트한다.** 오차단은 조용히 작업을 마비시키지만 미차단은 다른 방어선이 받는다.
- **저장소 전체를 훑는 일괄 변환을 손으로 돌리지 않는다.** 무엇이 바이너리인지는 git이 안다(`.gitattributes`) — `tr`·`sed`는 모른다(jar에서 `0x0D`를 지웠다). 손으로 해야 하면 대상을 화이트리스트로 열거하고, 그 전에 **"정말 고칠 게 있는가"를 `git diff`로 먼저 묻는다.** 광범위한 되돌리기(`checkout -- .`)도 파괴다.
- **복구 절차(백업·롤백)는 리허설로만 존재를 증명한다.** 쓰기 경로만 돌려보고 "있다"고 적지 않는다. 복원 드릴은 **일회용 격리 컨테이너**에서 돌리고, "복원됐다"가 아니라 **다음 기동이 성공할 조건**(스키마 이력 등)을 단언한다.
- **휴리스틱은 "1차 검증"을 기다리지 말고 이미 가진 golden으로 전수 측정한다.** fixture는 파서 검증용만이 아니라 하류 규칙의 시험대다. 개선 시엔 **before/after 전수 대조**로 "의도한 N건만 바뀌었음"을 증명한다.
- **우연히 옳은 코드는 다음 커밋에 틀려진다.** 요구사항 ID를 코드에서 전수 grep해 **참조 0인 ID**를 찾아라 — 그중 "이미 지켜지는 것"은 지금 테스트로 잠그고(RED가 없으니 **뮤테이션**으로 장치가 도는지 증명한다), "안 되는 것"은 보드에 올린다. 금지 목록은 키 이름이 아니라 **값**으로 검사한다(이름만 바꾼 우회를 잡는다).

### 계산·보정 규칙
- **상한(cap)이 있는 계산은 상한 없는 중간값을 만들지 않는다** — 포화 비교를 곱셈/누적 **전에** 스칼라로. `min(계산값, cap)`은 이미 터진 뒤라 보호막이 아니다. 포화 테스트 입력은 경계 바로 위가 아니라 극단값으로.
- **보정 규칙은 "얼마나 어긋났는가"로 원인을 갈라라.** 오차를 고치려는 보정이 원래 오차보다 큰 오차를 만들면 안 된다. 되돌릴 수 없는 필드일수록 보정에 보수적으로.
- **외부 시스템의 거절 신호를 재시도 가능한 오류로 분류하지 않는다** — 실패를 뭉개지 말고 `OK / 일시장애 / 거절(403·429)`로 나눈다. 일시장애만 백오프하고 거절은 중지+알림(SEC-08).

## 모듈 소유권 (동시 작업 충돌 방지)
> 두 개발자가 같은 저장소에서 병렬로 일한다. 이 규칙이 없으면 매번 "이건 내가 해도 되나"를 묻게 된다.

- `core/` = **상대 개발자**. `collector/`·`web/`·`scripts/`·`docs/`·`working-area/`·`.claude/` = 우리.
- core에 **신규 파일만 추가하는 additive 작업은 자율**이다(묻지 않는다). 커밋 전 `git status core/`로 **기존 파일 무수정을 증명**한다 — 수정된 파일이 하나라도 있으면 additive가 아니다.
- core **기존 파일 수정이 필요하면 멈추지 않는다.** `docs/91`에 "core 기존 파일이라 상대와 조율"로 적고 **우회해서 다음 일감으로 간다.**
- 예외: 상대가 `docs/91`·`progress-log`에 "우리가 안 한다"고 명시한 항목은 우리가 가져갈 수 있다.

## 작업 방식
- 커밋 메시지에 docs 요구 ID 참조(예: `BM-03`). `main`에 직접 커밋한다(브랜치를 만들지 않는다 — 1인용 저장소에 두 명이 직접 커밋하는 현행 방식).
- **loose-end 라우팅(기록강제)**: 열린 것은 "말로 끝내지 않고" 성격별로 정확히 **한 보드**에 즉시 적는다 —

  | 생긴 것 | 어디에 |
  |---|---|
  | 사람이 **정해야** 할 기획·정책·되돌리기 어려운 선택(기획 확정본과의 충돌 포함) | `working-area/decisions-needed.md` — **임의 확정 금지**, 확정본 임의 변경 금지 |
  | 잠정값으로 **진행 가능**한 기술 보류 | `docs/91-open-questions.md` — 잠정값+**재개 트리거** 명시, 가장 보수적(되돌리기 쉬운) 선택으로 진행하고 표시. 재개 트리거는 **"무엇이 참이 되어야 하는가"**로 쓴다 — 그때 떠오른 구현 수단을 적으면 다음 세션의 내가 그걸 요구사항의 제약으로 읽고 일감을 봉인한다(Q-50) |
  | **운영 배포** 시 사람이 할 것(시크릿·인프라·설정) | `working-area/pre-deploy-checklist.md` — `[필수]/[권장]/[완료]` |
  | **확정된** 결정(무엇을 왜) | `working-area/decision-log.md` |
  | 재사용 **교훈** | `docs/99-lessons.md` (위 교훈 축적 프로토콜) |
  | 외부 사이트 **실측** 발견 | `docs/98-field-notes.md` |
  | **무중단 중** 사용자가 알아야 할 것(한 일·자율 결정·블로커·다음) | `working-area/progress-log.md` — 매 배치 매듭 append, 최신 위, 지우지 않고 쌓기 |

- **refactor seam**: 미확정·잠정 결정은 "한 곳만 바꾸면 되는" 지점(인터페이스·상수·설정)에 격리해 나중에 최소 변경으로 뒤집을 수 있게 한다.
- 문서가 정답은 아니다 — 비합리적이거나 더 나은 방식이 보이면 임의로 바꾸지 말고 근거·대안을 제시해 확인받는다(Autonomous 모드 대상은 자율 반영+기록).

## Git & 커밋 규칙
- **커밋 타이밍**: 작업(기능) 단위가 끝날 때마다, **테스트 GREEN 확인 후** 커밋.
- **푸시는 사용자 지시가 있을 때만.** 평소엔 커밋까지만 하고, "푸시해"라고 할 때 에이전트가 `git push origin main`을 친다. **무중단 모드에서도 자동 푸시하지 않는다.** `allow` 목록에 없으므로 매번 승인 프롬프트가 사람의 마지막 확인이 된다.
- **force-push·원격 참조 삭제는 금지**(`--force`·`-f`·`--force-with-lease`·`--delete`·`-d`·`--mirror`·`--prune`·`+refspec`·`:refspec`). `.claude/hooks/guard.sh`가 강제한다 — `main`은 상대 개발자도 직접 커밋하므로 남의 커밋을 지운다. 일반 푸시는 non-fast-forward면 git이 알아서 거절하니 안전하다. 정말 필요하면 **사용자가 직접** 친다.
- **브랜치 전환·머지·리베이스는 사용자 지시가 있을 때만.** `git pull`도 머지다 — 지시받으면 하고, 충돌 후보를 먼저 대조해 보고한다.
- `.gitignore` 필수: `.env*`(텔레그램 봇 토큰·네이버 API 키 등 로컬 시크릿)·빌드 산출물·의존성 디렉토리.

## 빌드·테스트 명령
```bash
# 모듈별
cd core       && ./gradlew test        # 단위 + Testcontainers 통합
cd collector  && uv run pytest         # 전체(Docker 필요). 빠른 루프: -m "not integration"
cd web        && npm test && npm run build

# 전체 기동
docker compose up -d

# 종단·복구·게이트 (전부 CI가 돌린다. 격리 프로젝트/일회용 컨테이너만 쓴다)
bash scripts/smoke.sh              # 빌드→기동→web→nginx→core→postgres→파이프라인→기준가 (13단계)
bash scripts/backup.sh             # pg_dump + gzip + 7일 보관 + 오프사이트(설정 시)
bash scripts/restore-drill.sh      # 덤프를 일회용 컨테이너에 복원해 검증 (REL-04)
bash scripts/rollback-drill.sh     # V 전진 → R 역순 후진 → 재전진 (REL-05)
bash scripts/offsite-drill.sh      # MinIO에 대고 운영과 같은 업로드 경로 리허설
bash .claude/hooks/guard.test.sh   # 네트워크 차단 훅의 차단/통과 계약
bash .githooks/pre-commit.test.sh  # gitleaks 훅의 차단/통과 계약
```
- 테스트 DB: **Testcontainers(PostgreSQL 16)** — 통합 테스트 전용. 순수 도메인은 DB 없이 단위 테스트로 검증.
- 로컬 시크릿: 루트 `.env`(gitignore) — 텔레그램 봇 토큰·네이버 쇼핑 API 키·DB 비밀번호.
- **복구 절차(백업·롤백·오프사이트)는 리허설로만 존재를 증명한다.** 드릴은 전부 일회용 격리 컨테이너에서 돈다 — 운영/개발 DB에 절대 닿지 않는다.

## Autonomous(무중단) 모드 — 기본 운전 방식
> **기본값: 한 번 시작하면 "진짜 중요한 것" 말고는 멈추지 않고, 스스로 계획하며 계속 개발한다.**
> 적용 범위 = 테스트로 검증 가능한 전 구현(순수 도메인·어댑터·파서·마이그레이션·배선 등 GREEN으로 확인되는 코드).

**운전 루프**: 다음 막히지 않은 최고가치 증분 선택 → TDD Red→Green → GREEN 후 커밋 → 기록 → 진행 로그 append → (배치 매듭이면 비차단 보고) → 반복. 남은 일이 **전부** 막혔을 때만 사람에게 넘긴다.

- **증분 사이에 멈추지 않는다.** 커밋 후 곧바로 다음 증분을 **스스로 골라** 이어간다. "다음 뭐 할까 / 이대로 진행할까"를 묻지 않는다 — 증분 우선순위는 자율 실행 결정이지 기획 결정이 아니다.
- **스스로 계획한다.** 다음 일감은 남은 기획을 직접 읽어 정한다 — `docs/30-roadmap`(마일스톤 순서)·`docs/91-open-questions`(막히지 않은 잠정 항목)·`docs/90-planning-final`·`working-area/2nd-plan-intake`. 막힌 것(키·라이브·결정 대기)은 건너뛰고 코드로 갈 수 있는 최고가치를 택한다. 고른 근거는 `decision-log`나 task에 한 줄.
- **되돌릴 수 있으면 자율.** 되돌리기 쉬운 것(설계·리팩터 seam·개발 의존 추가·마이그레이션·additive 코드)은 **가장 보수적(되돌리기 쉬운) 기본값 + 기록**으로 진행하고 묻지 않는다. refactor seam으로 격리해 나중에 최소 변경으로 뒤집을 수 있게 둔다.

- **정지하는 것 = "진짜 중요한 것"뿐** (오직 이때만 멈춘다):
  - 데이터 파괴(운영 데이터·되돌릴 수 없는 삭제/덮어쓰기).
  - 외부 실행·실발송(텔레그램 실전송, 실사이트 크롤링, 유료·외부 API 실호출) — 훅 강제.
  - 비용 발생 / **보안 게이트를 넓히거나 끄는 변경**(예외 추가·차단 해제·인증 우회·시크릿 노출). 게이트를 **좁히거나 고정·추가**하는 것(버전 핀, 예외 축소, 새 검사 도입)은 자율 + 기록.
  - 기획 확정본(docs/90) 충돌, 또는 **되돌리기 어려운** 기획·데이터모델·정책 결정 — 임의 확정 금지. `decisions-needed`에 적되 **멈춰서 기다리지 말고 다른 막히지 않은 일감으로 계속 간다.** 남은 일이 전부 이런 것뿐일 때만 사람에게 넘긴다.
  - **force-push·원격 참조 삭제** — 훅 강제. (일반 `git push`는 정지조건이 아니다. 다만 **사용자 지시가 있을 때만** 친다 — 자동 푸시는 하지 않는다.)
- **묻는 것은 정지조건일 때뿐이다.** `AskUserQuestion`은 위 목록에 해당할 때만 쓴다. 그 밖의 불확실(구현 방식·수치 선택·순서·범위)은 **가장 보수적(되돌리기 쉬운) 기본값 + 보드 기록**으로 진행하고 계속 간다. "묻지 않는다"는 산문은 도구 앞에서 무력하므로 도구 이름으로 못박는다.
- **한 턴 안에서 계속 간다.** 이 도구에서 "채팅에 글을 쓰고 도구 호출을 멈추는 것"은 **턴을 끝내는 것**이고, 그러면 사용자가 다시 말하기 전까지 아무것도 못 한다. 그러니 증분마다 보고하고 멈추지 않는다. **턴을 끝내는 조건은 셋뿐**: ① 정지조건에 닿았다 ② 막히지 않은 일감이 하나도 없다 ③ 컨텍스트가 한계에 가깝다. 그 외에는 커밋 직후 **곧바로 다음 증분의 첫 도구를 호출**한다.
- **배치 = 위 ①②③에 닿을 때까지.** "증분 두세 개 하고 한 번 보고"가 아니다.
- **보고는 파일에.** 상세는 매 증분 `working-area/progress-log.md`에 append(한 일·커밋·자율 결정·⚠️결정필요/블로커·다음). **채팅 보고는 턴 마지막에 한 번**, 짧게. 진행 중 알림이 필요하면 도구 호출 사이에 한두 문장만.
- **그래도 지키는 것**: 테스트 GREEN 후에만 커밋 / 기록 의무(`progress-log`·`decisions-needed`·`docs/91`·`docs/99`·`decision-log`) / 확정본 임의 변경 금지.
- 정지조건 중 **결정론적인 둘(파괴적 `git push`·실 네트워크 호출)은 훅으로 강제**된다(`.claude/hooks/guard.sh`, PreToolUse, `matcher: "Bash|PowerShell"`). 나머지("임의"의 판단이 필요한 것)는 이 지침의 몫이다. **훅은 도구 이름으로 스코프된다** — matcher에 없는 도구로 같은 명령을 보내면 아무것도 막지 않는다(2026-07-10 실측: `PowerShell` 도구가 그랬다).
