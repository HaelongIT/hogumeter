# 보안·운영 노트 (SECURITY)

hogumeter는 **1인용 개인 프로젝트**다. 아래는 시크릿 취급·플랫폼 접근 윤리·데이터 처리에 대한 운영 기준이다.

## 시크릿 취급
- 모든 로컬 시크릿은 루트 `.env`에 둔다 — **`.env*`는 `.gitignore` 대상, 절대 커밋 금지.**
- 형식은 [`.env.example`](.env.example) 참조: `DB_PASSWORD` · `TELEGRAM_BOT_TOKEN` · `NAVER_CLIENT_ID`/`NAVER_CLIENT_SECRET`.
- 시크릿이 커밋에 섞이면 즉시 회전(토큰 재발급) 후 히스토리 정리.
- `docker-compose.yml`은 값이 아니라 `${VAR}` 참조만 담는다. Postgres 포트는 `127.0.0.1`로만 바인딩(외부 미개방).

## 플랫폼 접근 윤리 (핵심 원칙 5)
데이터 수집은 아래 잣대를 **자동 적용**한다. 새 플랫폼 추가·차단 변화 시 이 잣대로 판정한다.
1. **공식 API 우선** (네이버 쇼핑 등).
2. **기술적 차단이 있으면 우회하지 않는다** — 프록시·UA 위장·IP 회피 금지. (예: 당근 IDC 대역 차단 → 자동 fetch 시도 안 함.)
3. **차단 없는 공개 페이지는 저빈도·개인용으로 수용** — 게시판당 **1req/min**, robots 존중. 약관 회색지대임을 인지한다.

> 커뮤니티 크롤링은 약관 회색지대다. **1인용·저빈도**라는 전제에서만 수용하며, **서비스화 시 데이터 계약(약관·쿼터·법무) 재검토가 필수 게이트**다([`docs/90-planning-final.md`](docs/90-planning-final.md) §12 리스크 장부).

## 데이터 처리
- 크롤링 원본은 PostgreSQL `JSONB`에 **보관 전용**으로 저장(정규화된 값은 별도 컬럼/EAV 테이블). JSONB 남용 금지.
- 개인정보를 능동 수집하지 않는다 — 공개 게시글의 가격·링크·반응 신호 위주. 중고 매물의 판매자 설명은 승격 시 1회만 확보.
- 사기·최종 판단 로직을 만들지 않는다(원칙 2). 위험 신호는 나열만, 판정은 사람.

## 배포 전 사람이 할 일
운영 배포 시 시크릿·인프라·설정 갭은 [`working-area/pre-deploy-checklist.md`](working-area/pre-deploy-checklist.md)에 `[필수]/[권장]/[완료]`로 관리한다. 실 발송(텔레그램 실전송)·외부 API 실호출·비용 발생은 사람 승인 게이트.

## 취약점 보고
1인용 개인 프로젝트라 공식 보안 창구는 없다. 이슈나 리포지토리 오너에게 직접 알리면 된다. 시크릿 유출 정황을 발견하면 공개 이슈로 올리지 말고 비공개로 알릴 것.
