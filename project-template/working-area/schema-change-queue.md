# 스키마 변경 큐 (Schema Change Queue) — 옵션

> **운영 DB를 직접 못 만지는 환경**(DBA 소관·read-only)에서 필요한 DB/외부 스키마 변경을 모은다.
> 운영은 건드리지 말고 여기 DDL을 적재하고, **동시에 로컬 테스트 스키마엔 같은 변경을 반영**해 테스트가 돌게 한다.
> 운영 반영은 담당자 몫. 적용되면 `[적용 완료]` + `decision-log`에 기록.
>
> (운영 DB를 직접 만지는 프로젝트면 이 파일 삭제.)

상태: **[적용 필요]** 운영 대기 / **[적용 완료]** / **[예정]** 해당 기능 시점에 추가.
