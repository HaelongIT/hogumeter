# 배포 전(go-live) 체크리스트 — 운영을 막는 것들

> **이 문서 = 운영 배포 단일 점검 지점(허브).** 로컬/테스트에선 되지만 **운영에서만 별도 설정·적용이 필요한 것**을 한곳에.
> 개발 중 "운영에서 따로 해줘야 한다"가 드러나면 **그 자리에서** `[필수]/[권장]/[완료]`로 추가한다(말로 끝내지 않음).

## A. 데이터베이스
- **[필수]** 운영 DB 스키마 반영 — `working-area/schema-change-queue.md`의 `[적용 필요]` 전부. 미적용 시 무엇이 안 되는지 명시.

## B. 시크릿 · 환경설정
- **[필수]** 운영 시크릿({{PROD_SECRETS}}) — DB 접속·토큰 서명키·메일 계정 등. git/CI에 평문 금지.
- **[필수]** 유출 시크릿 회전 — git history에 평문이 올라간 적 있으면 새 값으로 교체(외부 재발급 포함).

## C. 보안 · 네트워크
- **[필수]** CORS 허용 오리진 = 운영 도메인. 리버스프록시(same-origin·헤더 전달) 설정.

## D. 빌드 · 배포
- **[필수]** 테스트 GREEN 후 빌드. 운영 프로파일/환경변수 확인. 프론트/백 계약 변경분은 **동시 배포**.
- **[권장]** CI/CD 파이프라인 — 테스트·빌드 자동화. CI 시크릿은 평문 커밋 금지(§B와 연결).

## E. 인프라 의존성
- **[필수]** {{INFRA}} 가동 확인(캐시·메일 등). 파일 저장 경로/권한·볼륨 마운트(재배포 유실 방지).
