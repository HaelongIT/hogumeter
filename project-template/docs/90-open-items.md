# 90. 보류 보드 (Open Items) — 기술 보류

> **단일 보드.** 잠정값으로 진행 가능한 한계·미결·임시방편·추후과제를 여기 모은다.
> 매 세션 시작 시 읽어 열린 항목을 인식한다. **말로 끝내지 않고 그 자리에서 적는다.**
> 각 항목에 **재개 트리거**(무엇이 충족되면 처리)를 반드시 남긴다. 해소되면 `working-area/decision-log.md`로 이관하고 여기서 제거.

## B-1. <제목> — ⏸ 보류
- **현상**: 무엇이 미완/임시인가.
- **왜 보류**: 지금 안 해도 되는 이유(위험>가치·조건부·저영향 등).
- **잠정값 / seam**: 지금 어떻게 진행하나 + 되돌릴 수 있는 단일 지점.
- **재개 트리거**: 무엇이 충족되면 처리하는가.
