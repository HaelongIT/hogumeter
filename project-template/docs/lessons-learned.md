# Lessons Learned — 교훈·트러블슈팅 누적

> **TDD·버그 수정·리팩토링·트러블슈팅**에서 얻은 **재사용 가능한 교훈**을 아래 형식으로 append.
> 매 세션 시작 시 먼저 읽어 같은 실수를 반복하지 않는다.
> 버그·리팩토링 항목은 교훈에 **"원인 → 해결"을 명시**한다(런타임/설정/환경 함정 포함).

**기록 대상**: 비자명한 테스트 실패 원인·해결 / 버그·리팩토링 결함(원인→해결) / 반복 픽스처 / 명세 빈틈 / 라이브러리·버전 함정.
**기록 금지**: 한 번만 쓰는 정보, 코드에 이미 드러난 사실, 일반 상식.

## 형식
```
### YYYY-MM-DD · <한 줄 제목>
- **맥락(Context)**: 어떤 작업 중이었나
- **교훈(Lesson)**: 무엇이 비자명했나 — 버그/리팩토링이면 "원인(root cause) → 해결(fix)"
- **적용(How to apply)**: 다음에 같은 상황에서 어떻게
- (선택) **명세 영향**: 갱신이 필요한 docs
```
