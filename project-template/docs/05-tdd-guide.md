## 1. 워크플로우 (Red → Green → Refactor)
1. **Red** — 실패하는 테스트 먼저(모듈 문서의 Given-When-Then 인수조건을 1:1 변환).
2. **Green** — 통과하는 최소 구현.
3. **Refactor** — 중복 제거·구조 개선. 테스트는 계속 GREEN.
