# 2차 기획 intake

## A. 큰 그림
- 2차 기획은 1차가 커버하지 못한 **구매 이후 루프와 판단 보조 표면**을 추가한다: 구매 기록·사후 성적표(호구였는지의 최종 판정), 신호등(지금 잡을 딜 유무의 압축), 딜 주기(기다림의 기대치), 딜 보관함(고민 구간), 주간 다이제스트(흐름 요약), 우선순위 맥락(대기열 상기). 전부 신규 수집 0 — 1차가 만드는 데이터의 재조합이다.
- 부산물로, 신기능 감사 과정에서 1차의 암묵 가정·미정의가 다수 발견되어 **1차 문서 소급 정정**(시간 좌표계, 집합 명명, 이상치 판정 시점 등)이 함께 들어온다.
- 1차 대비 방향 변화: **없음 — 기능 추가 위주.** 6대 원칙·스택·플랫폼 잣대 불변.
- 새로 들어오는 기능/축:
  - 기능1(2차): PUR — 구매 기록·스냅샷·관찰 모드·사후 성적표
  - 기능2(2차): SIG — 신호등("지금 잡을 딜이 있는가")
  - 기능3(2차): CAD — 딜 주기 통계
  - 기능4(2차): WATCH — 딜 보관함(핀·결말·회고) ※사용자 유보
  - 기능5(2차): DIGEST — 주간 요약 리포트
  - 기능6(2차): PRI — 우선순위 맥락 ※존폐 승인 계류
  - 공통 축: 세 집합 명명 / 시간·관측 좌표계 / 가격 역할 3분법 / 판정 생애주기·배치 / 알림 공통 원칙

## B. 항목별 상세

### [B-1] 세 집합 명명 계층 (pricingSet / occurrenceSet / signalSet)
- 성격: NEW + 정리
- 닿는 곳(추정): docs/00-overview.md 용어집 · 신규 문서 제안 docs/03(02와 10 사이, "딜 집합·시간 좌표계") · docs/11
- 확정된 것:
  - pricingSet(값의 통계: 기준가·P25·성적표) = CONFIRMED + 키워드 무히트 + 선택 축값/basis 일치 + 이상치 미해당(승격 LOWER는 포함) + 배치유보 제외
  - occurrenceSet(시간의 통계: 딜 주기) = 정체성 축만 — CONFIRMED + 키워드 무히트 + 선택 축값 일치 + UPPER 제외(가격으로 감지된 정체성 실패 프록시) + LOWER는 기각만 제외 + 배치유보 포함
  - signalSet(지금의 단정: 신호·상시 표시) = CONFIRMED + 키워드 무히트 + **무경고(⚠️라벨 제외)** + 비이상치 + ENDED 아님 + 신선도 자격 + 선택 축값. 트리거(목표가/paidPrice) 무관
  - 집합×창은 독립 차원 / 집합별 tier 독립 / 모든 통계는 자기 집합·자기 분모 명시
  - 1차 문장의 집합 동사("제외/포함/배제") 목적어 매핑 표(동사 재조준 표) 동봉
- 아직 열린 것: 없음
- 왜: "유효 딜"이라는 단일 어휘가 세 질문(값/시간/지금)을 겸하며 오독·오구현을 유발 — 명명으로 종결.
- 원문 근거: CAD 정밀 5R(명명), 6~7R(UPPER/LOWER 정밀화), SIG 7R(술어 닫힌 목록).

### [B-2] 시간·관측 좌표계 (firstSeen 불변 / lastEvidenceAt / observedFrom / 관측시계)
- 성격: 수정 + 정리 (일부는 C 후보)
- 닿는 곳(추정): docs/02-domain-model.md · docs/11 BM-06 · docs/00 핵심수치
- 확정된 것:
  - firstSeen = **발생 시각**(라이브도 글 postedAt 우선, 파싱 실패 시 첫 관측 폴백 / 백필 = postedAt, 날짜만이면 23:59 KST). 생성 시 1회 확정·불변(재분류·후행 postedAt 확보로 소급 변경 없음). capturedAt(수집 시각) 별도 보존. firstSeen 독자 목록: 기준가 기간 필터/CAD/PUR 산입·as-of/병합 윈도우/lastEvidenceAt 성분
  - lastEvidenceAt = max(최신 병합 글 firstSeen, 마지막 PRICE_CHANGED) — "살아있음의 적극 증거"(단순 생존 관측 불포함). 스칼라
  - observedFrom(variant별) = min(백필 성공 사이트 도달 시점) 또는 등록 시각. **유효 창 규칙**: 모든 시간 통계 = 대상 창 ∩ [observedFrom, now/발급], 짧으면 "관측 범위 N개월" 표기
  - 관측시계: 신선도 staleness = (소스 사이트 마지막 성공 폴링) − lastEvidenceAt — 수집 공백 시 정지. 시간 통계의 내부 공백은 caveat 수용/실시간 판정은 시계 정지(처리 차이 병기)
- 아직 열린 것: postedAt 파싱 가능성 실측(M0 스파이크, 라이브 포함).
- 왜: "부재 = 정보 vs 무지"의 구분 축이 1차에 없어 시간 통계·신선도가 관측 공백을 딜 부재/노화로 오독.
- 원문 근거: CAD 정밀 2R(불변), 8R(observedFrom), 통합 8R(관측시계).

### [B-3] 가격 역할 3분법 (priceFirst / priceLast / priceMin)
- 성격: 정리 + 수정
- 닿는 곳(추정): docs/02 · docs/16(신규)·15(신규)의 각 소비처
- 확정된 것:
  - priceFirst = 발생·분포(기준가, 성적 percentile — 1차 유지) / priceLast = "지금"(신호 색, 지금 표시, 비교표, 다이제스트 활성 딜) / priceMin = "지나간 기회"(회고, 다이제스트 종료 딜 = 생애 스칼라 / 성적표 "기간 내 최저 기회" = 관찰 **만료 시점 동결** min, 지각 백필분은 capture 시점 스칼라 / 관찰 문맥 N) / priceMax = 역할 없음·참조 금지
  - 성적표 출력 성분별 잣대 분리: percentile=priceFirst, "기간 내 최저 기회"=priceMin(라벨 교체 포함), 병존 각주
- 아직 열린 것: 없음
- 왜: "딜 = 단일 가격" 암묵 가정이 신호(과거 가격으로 점등)·성적표(기회 과소평가 = 성적 부풀리기) 오류를 유발.
- 원문 근거: SIG 정밀 4R, WATCH 7R, 전체 통합 1~4R.

### [B-4] 이상치 판정 생애주기 + 배치 판정
- 성격: 수정 + 정리 (일부는 C 후보)
- 닿는 곳(추정): docs/11 BM-05 · docs/02 · docs/benchmark/ 하위(미상)
- 확정된 것:
  - outlierFlag = **유입 시 1회 평가 후 영속**, 분포 드리프트 재평가 없음. 전 시스템 단일 판정 원천(신호·알림·기준가·공시·성적 동일 플래그). 변경 서열: 사람 > 배치 > 잠정(사람 플래그 불가침)
  - LOWER 3상태: 미확정(pricing 제외·occurrence 포함·🔥 알림) / 승격(전 집합 포함) / 기각(전 집합·공시 제외, 이력 열람만). 🔥 quiet 관통 = LOWER 전체(확정·잠정·"참고용" 무관)
  - SPARSE/NONE 폴백 컷의 현재가 조회 = 유입 시 이벤트 구동 1회 허용. 조회 실패 = PROVISIONAL(근거 부재)
  - 배치: 백필 딜 = PENDING_BATCH(판정 유보·occurrence만 포함) → 배치 완료 이벤트(전 사이트 종결)에 완전 분포로 일괄 판정. 백필 중 라이브 딜 = PROVISIONAL(폴백 컷 잠정, 전 집합 잠정 적용) → 완료 시 최종. 완료 이벤트 처리 순서 [일괄 판정→유예 스냅샷 확정→발급 진입]. PROVISIONAL 해소 4경로(현재가/배치/tier 승격/사람). 하향 확정 후속 알림 없음(보류분은 플러시 재평가)
  - 백필 진행 = "과거 딜 수집 중" 배지(상세·기능4·다이제스트)
- 아직 열린 것: 이상치 컷 수치(±IQR 배수 등 — M0 기존 위임).
- 왜: 판정 시점 미정의가 화면 간 모순·크롤링 순서 의존 판정을 낳음.
- 원문 근거: SIG 정밀 5R, 통합 교차 1~6R.

### [B-5] 알림 체계 공통 원칙 (발송 단위 / 계열 표 / 라이브 재평가 / 비대칭)

- 닿는 곳(추정): docs/12-feature-monitoring-alerts.md (계열 구조 표 신설 제안)
- 확정된 것:
  - **발송 단위 원칙**: "관측 대상(딜 이벤트·중고 매물)의 상태 사건당 발송 1통" — 복수 사유 충족 시 최고 우선 프레이밍 + 나머지 본문 병기. 서열: 트리거 계열(🔥>특가>목표가>괜찮은 딜>관찰) > 후속 계열. 1차 "최고 강도 1발"은 이 원칙의 트리거 부분집합으로 재배치
  - 계열 구조: 트리거 / 후속(검증·가격변동·종료·확인 필요·부활 — 인상 특례 2종: 알림 나간 딜=무조건, 핀 딜=1회) / 정기 리포트(다이제스트 — 스케줄 구동 별축, 관할 각주) / 병기 사유(본문 순서 규격 [충족조건→검증상태→참고줄])
  - **라이브 서술 재평가 원칙**: quiet 보류분 플러시 시 본문의 모든 라이브 서술은 발송 시점 재평가(갱신·제거 O / 신규 추가 X). 기존 "종료·가격" 재확인은 특례로 흡수, 판정 상태 재확인 추가(하향 시 강도 재산정)
  - **비대칭 원칙**: 알림 = 재현율(후보 톤 발화 허용) / 상시 표시 = 단정 금지(signalSet만)
- 아직 열린 것: 참고줄 성분은 [PRI-계류] — 기능6 드랍 시 병기 목록·본문 규격에서 해당 성분 삭제.
- 왜: 신기능들이 알림 밀도를 높이며 1차 1발 원칙의 적용 단위(트리거 간)가 좁았음이 드러남.
- 원문 근거: WATCH 9R(발송 단위), BUDGET 4R(라이브 재평가), 전체 통합 4R(중고 일반화).

- 원문 근거: 각 기능 정밀 라운드+통합 교차 전체.

## C. 확정본 충돌 후보
- **C-1 [B-10 연동] DealEvent 상태기계에 ENDED→ACTIVE 재개 전이 신설**: 확정본·02의 상태기계는 단방향(NEW→ACTIVE→VERIFIED→ENDED). "종료 후 가격 재개" 관측을 받을 전이가 없어 부활·재핀·신호가 공허해짐. 빈칸 보완 성격이나 봉인된 상태기계의 변경이므로 사람 확인 필요.
- **C-2 [B-2] firstSeen 의미 재정의(첫 관측→발생 시각, postedAt 우선)**: 확정본의 백필 as-shown 수용과 정합하나, 라이브 딜까지 postedAt 우선으로 통일하는 것은 원문 범위 초과.
- **C-3 [B-2] K_fill 자동 확장 상한 = min(12개월, observedFrom)**: 확정본의 "확장 상한 12개월"을 관측 범위로 추가 제약(커버리지 초과 확장 금지).
- **C-4 [B-4] 이상치 판정 = 유입 시 1회·영속(드리프트 재평가 없음)**: 확정본 BM-05는 산식(median/IQR)만 정의 — 판정 시점·영속성은 해석 확정이라 원문 의도 확인 필요.
- **C-5 [B-1 연동] 제외 키워드 라벨(⚠️) 모드 = 모드 무관 전 통계 제외(가시성만 차등)**: 확정본은 라벨 모드의 표시만 정의, 통계 지위는 미정의 — "포함"으로 읽어온 해석이 있었다면 배치됨.
- **C-6 [B-13] 미선택 수요축 값의 전 집합 제외**: 확정본 REG-02는 선택의 통계 효력 미정의 — 이 확정이 기준가 표본을 좁힐 수 있어 사람 확인 권장.

## D. 폐기/보류한 아이디어
- **BUDGET 돈 산술 코어(예산 풀·잔여·빠듯 판정) — 드랍**: 경계 술어 "두 제품 이상의 금액을 결합하는 모든 산출". 재검토 조건: 등록 제품 수십 개 규모 + 기간 예산 명시 운용 패턴 확인 시(그때도 산술 아닌 표시만). 우선순위 정렬(b)·맥락(c)은 B-12로 분리 생존.
- **신호 이력 테이블**: 전환 계산용 시계열 저장 기각 — 직전 발송 시점 1벌 저장으로 대체(B-11).
- **자격 상실 시 MISSED 추정 자동 전이**: "살아있다고 단정 못 함"을 사망 판정으로 오용 — 확인 필요 알림+수동 결말로 대체(B-10).
- **가격 변경 이력 영속(창 내 min 재생용)**: 소비처 1개 대비 과잉 — 만료 시점 스칼라 동결로 대체(B-3).
- **중고 구매 성적표 / 다이제스트 중고 섹션**: 중고 기준가 무합성(1차 결정)의 귀결로 불가·기각. Phase 3 재검토 여지.
- **참고 줄에 기준가·경과일 병기 / 동일 참고 줄 축약형**: 본문 비대·이중 분모·구현 대비 가치 0으로 각각 기각(B-12 신호 1자만 채택).
- **유머 등급 라벨(호구 지수 B+ 등)**: 라벨이 숫자를 가려 정직성과 충돌 — 전 기능 공통 배제.
- **핀 대상의 중고 매물 확장**: 중고 고민은 기능5의 평가기·메모·비교가 관할 — 경계 명기로 기각.
