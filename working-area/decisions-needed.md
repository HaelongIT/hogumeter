## D-7. 네이버 쇼핑 API 서비스 종료(2026-07-31) — 현재가 데이터 소스 대체 방안

**무엇이 걸렸나**
- 네이버 개발자센터가 검색 **쇼핑/책/전문자료 API**를 **2026-07-31(금)** 전면 종료한다고 공지(사용자가 공지 원문 확인, `docs/91` Q-3 참조). 대체 API 안내 없음.
- `CurrentPriceProvider`(BM-06 currentPrice — 갭 계산·콜드스타트 잭팟의 유일한 입력), REG-01 후보 검색, CMP-01 네이버 leg가 전부 이 API를 전제로 설계돼 있었다. 키를 지금 발급받아도 일주일 뒤 죽으므로 "키 발급 대기"는 더 이상 유효한 계획이 아니다.
- 대안마다 절대 원칙 5(공식 API 우선·기술적 차단 우회 금지·차단 없는 공개 페이지는 저빈도·개인용 수용)의 해석이 갈려서 임의로 못 정한다.

**지금 상태**: 코드 변경 불필요 — `StubCurrentPriceProvider`가 이미 null을 반환하고, 도메인은 그걸 "현재가 미확립"(Q-53)으로 정직하게 처리한다. 급하게 막힌 건 없다.

**선택지(참고용 — 아직 아무것도 권고하지 않음, 사용자가 별도로 검토 중)**
- 네이버쇼핑 검색결과 페이지를 저빈도·개인용으로 긁는다(원칙 5의 "차단 없는 공개 페이지" 해당 여부 검토 필요)
- 다른 가격비교 공개 API(다나와·에누리 등)로 대체
- 쿠팡 크롬 확장(CMP-02)만으로 현재가를 대신한다 — 등록 제품마다 쿠팡 링크가 있어야 함
- 현재가 없이 운영 — 갭·콜드스타트 잭팟 기능을 포기하고 기준가·사례 중심으로만 판단

**연결**: `docs/91` Q-3(업데이트됨), Q-79(CMP-01 네이버 leg).
