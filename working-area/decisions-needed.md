## [열림] D-2. Spring Boot 3.x 유지 vs 4.x 이관 (지원 종료 임박)
- **배경**: 스택 확정본은 "Spring Boot 3.x (확정, 변경 금지)"인데, Boot **3.5.x의 OSS 무상 지원 기간이 2026년 중 종료**(커밋 시점 3.5.16 사용). Boot 4.0은 2025-11 GA. 즉 3.x 고정은 무상 보안패치 종료 리스크를 안고 가는 선택.
- **결정 질문**: (1인용·개인 배포이므로) 3.5 그대로 유지할지, M1 착수 전 4.x로 올릴지. **스택 확정본과 충돌하므로 임의 변경 불가 — 사람 결정.**
- **선택지**: (a) 3.5 유지 — 이식 비용 0, 개인용이라 보안 리스크 수용 가능(외부 노출 최소). (b) 4.x 이관 — 최신 지원, 단 확정본 개정 + 의존성·API 변경 검토 필요. Java 21은 둘 다 OK.
- **현재 잠정값**: (a) 3.5.16 유지(가장 되돌리기 쉬움 — seam은 build.gradle.kts 버전 1줄).
- **영향**: `core/build.gradle.kts`, 확정본 스택 섹션, CLAUDE.md 스택.
