# 1차 검증 런북 — M1 코드 완결 후 실 데이터로 검증하기

> M1 알림 경로 **코드는 완결**됐다(등록→수집→기준가→판정→실 발송→미상 큐 버튼→무시→사후학습).
> 남은 것은 **실 데이터로 도는지** 보는 것뿐이고, 그건 사람의 시크릿·승인·관찰이 있어야 열린다.
> 이 문서는 그 순서를 못박는다. 상세 게이트는 `pre-deploy-checklist.md`, 완료 기준은 `docs/30-roadmap.md`.

## 0. 사전 준비 (사람이 발급·승인)

| 무엇 | 왜 | 없으면 |
|---|---|---|
| **텔레그램 봇 토큰** (@BotFather) + 내 chat_id (`getUpdates`) | 알림 실 발송·인라인 버튼 | 알림이 로그 스텁으로만 (스텁이라 실제로 안 감) |
| **네이버 쇼핑 API 키** (선택, Q-3) | 현재가·갭·후보 검색 | 현재가 미확립 → 갭 미표시(정직하게 "미확립") |
| **robots 실 대조** `ALLOW_REAL_ROBOTS=1 bash scripts/check-robots.sh` | 실 폴링 전 3사 robots 확인(§F) | 차단·과호출 위험 |
| **D-3 결정** (차단 재개 경로) | 차단당하면 되살릴 방법 | 차단 후 수동 개입 필요 |

## 1. `.env` 채우기 → 점검

```bash
cp .env.example .env            # 값 채움 (DB_PASSWORD 필수; 텔레그램·네이버는 위 0에서)
bash scripts/preflight.sh dev   # 로컬 검증. 공개 배포면 prod (web 인증 필수 검사)
```
`preflight`가 **조용히 오작동할 설정을 미리 잡는다** — 공개인데 web 인증 없음(데이터 노출)·텔레그램 켰는데 토큰 없음·실 폴링인데 robots 미대조. `PREFLIGHT OK`가 나와야 다음으로.

## 2. 기동 → 종단 스모크

```bash
docker compose up -d
bash scripts/smoke.sh           # 빌드→기동→web→nginx→core→postgres→파이프라인→기준가 (13단계 + 알림·큐·카운터 마커)
```
`SMOKE PASS`면 스택이 종단으로 산다. 이 시점엔 아직 **스텁 데이터**(실 폴링 전).

## 3. 제품 등록

- web(또는 REST)으로 **아이폰 17 일반 256GB** 등록. 알림 정책(목표가·기간·K·제외 키워드)도 설정.
- `TELEGRAM_ENABLED=true`면 알림 정책 화면이 "실 발송 중"을, 아니면 "로그로만 남습니다"를 정직하게 표시한다.

## 4. 실 폴링 켜기 (승인 사항)

```bash
# 0의 robots 대조를 돌린 뒤에만:
#   .env에 COLLECTOR_ALLOW_NETWORK=1  →  docker compose up -d
```
이제 collector가 핫딜 3사를 실제로 폴링한다. 저빈도·개인용(플랫폼 잣대 5).

## 5. 관찰 — M1 완료 기준 (docs/30)

- [ ] **기준가 형성**: 실 딜이 쌓이며 아이폰 17 256GB 기준가가 뜨고, **운영자 체감과 부합**하는가.
- [ ] **알림 수신**: 특가/목표가 딜에 텔레그램 알림이 오는가(강도·갭·조건 태그·[🔕무시] 버튼 포함).
- [ ] **버튼 분류**: 미상 큐 알림의 [승격]/[기각]이 실제로 처리되는가(web·텔레그램 둘 다).
- [ ] **사후학습 수렴**(1~2주): 오알림을 [무시]하면 빈출 키워드가 KEYWORD_SUGGEST로 제안되고, 제외 키워드로
      추가하면 그 노이즈가 줄어드는가.
- [ ] **품절 오알림 없음**(Q-27③): 이미 끝난 딜에 "지금 사라"가 안 가는가.
- [ ] **관측**: `docker logs`의 틱 카운터(`matched[...] firstAlertsSent heldFlushed stepsFailed` 등)가 침묵/드리프트를 드러내는가. `heartbeat`가 3틱 연속 실패면 관리 알림이 오는가(OBS-03).

## 6. 실패 시

- 스택이 안 뜸 → `GET /api/v1/health`(어느 컴포넌트가 죽었나) + `docker logs`.
- 알림이 안 옴 → `/api/v1/alerts/status`의 `delivering`(스텁이면 false) + 정책이 저장됐는지.
- 파서 정확도 이슈(가격·배송비·품절) → `docs/91` Q-63~65·Q-46(실 표본으로 튜닝, 골든 전수 대조).
